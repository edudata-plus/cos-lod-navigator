# 学習指導要領LODナビゲーター

検索で到達した細目を、学習指導要領本文全体の階層と前後関係の中に戻して理解するための、GitHub Pages向け静的Webアプリです。

- 検索結果は、小学校 → 中学校 → 高等学校の順にまとめ、各学校種内では `qb:order/rdf:value` 順に表示します。

## v0.7.1 の主な実装

- 検索結果に含まれる本文をキャッシュから即時表示し、周辺文脈のみ非同期で構築
- 選択細目の直下にある子項目を専用SPARQLで取得
- 子項目は15件単位で追加表示
- 検索結果をSPARQL側・ブラウザ側の両方で `qb:order/rdf:value` 順に整列

- 対象を `cs:courseOfStudy` が小学校2017・中学校2017・高等学校2018を指す細目に限定
- 学校種・教科・科目・学年のフィルタ値にRDF値（URIまたはデータ値）を使用
- 検索語なしで学校種だけを選んだ検索に対応
- 検索結果を100件単位で取得し、「次の100件を表示」でページング
- 本文・コード・URI・教科名・科目名を含む検索用文字列を `COALESCE` 付きで生成
- 細目選択時に、検索クエリとは別に祖先系列を再帰取得
- 各階層の兄弟項目を `qb:order/rdf:value` 順に取得
- 選択細目の前後3件を初期表示
- 省略項目を「前の項目／続きの項目を15件まで表示」で追加取得
- 展開トグルと項目選択を別ボタンに分離
- 接続失敗時は自動的にサンプルへ切り替えず、エラーと再取得操作を表示
- 中央列は「検索結果／本文表示／学びのつながり」のタブ構成

## ファイル構成

```text

app.js              検索・本文・周辺文脈の画面制御
sparql-queries.md   クエリ設計の説明
specification.txt   実装時点の仕様書
README.md           このファイル
```

## GitHub Pagesでの公開

1. フォルダ内の全ファイルをリポジトリ直下へ置きます。
2. GitHub の Settings → Pages で公開元ブランチを指定します。
3. PagesのURLへアクセスします。

サーバーサイド処理はありません。ブラウザから公開SPARQLエンドポイントへ問い合わせます。

## 調整しやすい箇所

`config.js` で、次を変更できます。

- `endpoint`: SPARQLエンドポイント
- `examples`: 検索例
- `searchPageSize`: 検索結果の1回の取得件数
- `contextInitialLimit`: 周辺文脈の初期表示件数の目安
- `contextMoreSize`: 「続きを表示」1回の追加件数
- `nearbyWindow`: 選択項目の前後に表示する件数
- `targetCourseOfStudies`: 対象とする学習指導要領URI

クエリを変更するときは `sparql.js` を編集します。UI制御とは分離してあります。

## 補足

学年については、エンドポイントが返すRDF値をそのまま選択肢の値として利用します。URIの場合はIRI、リテラルの場合はリテラルとしてSPARQLへ埋め込みます。

## v0.7.3 の変更点

- 周辺文脈内の兄弟項目・上下項目・遅延取得した子項目に、独立した展開ボタンを追加しました。
- 展開時に初めて直下の子項目を取得する遅延ロード方式です。子項目は15件ずつ追加表示できます。
- 項目本文を選択しなくても、その場で下位階層を閲覧できます。
- 本文表示では、選択細目の `schema:description` を本文として一度だけ表示します。
